## 2025-03-29 - [Added visual feedback for login submission and focus styles for keyboard navigation]
**Learning:** Found that the primary login form lacked visual feedback upon submission, creating a potential for double-clicks, and the custom CSS lacked a `:focus-visible` outline for interactive elements, hindering keyboard navigation accessibility.
**Action:** Always provide immediate visual feedback (e.g., loading spinner and disabled state) for primary form submissions even for fast local redirects to make the action feel intentional, and consistently include `:focus-visible` styles with sufficient contrast for links, buttons, and inputs.
